Hoisting in JavaScript

Hoisting is the process of moving variable and function declarations to the top of their scope before the code is executed. This behavior can affect how code runs and may lead to unexpected results if not understood properly.

Example 1: Hoisting with var for variable declaration

When using var, the variable is hoisted and initialized with undefined, but its assignment is not hoisted.

            
              console.log(number); // undefined, as the declaration is hoisted but not the assignment
              var number = 10;
              console.log(number); // 10

Example 2: Hoisting with let and const for variable declaration

Variables declared with let or const are hoisted but not initialized, leading to a ReferenceError when accessed before declaration.

            
              console.log(number); // ReferenceError: Cannot access 'number' before initialization
              let number = 10;     // or const number = 10;
              console.log(number);  // 10

Example 3: Temporal Dead Zone with let and const

The "Temporal Dead Zone" is the region where the variable is hoisted but cannot be accessed before its declaration. This applies to both let and const.

            
              console.log(number); // ReferenceError: number is in the TDZ (Temporal Dead Zone)
              let number = 10;
              console.log(number);  // 10

Example 4: Undeclared Variables

Undeclared variables do not exist until code assigns them a value. Assigning a value to an undeclared variable implicitly creates it as a global variable.

            
              function hoist() {
                a = 20; // Implicit global variable
                var b = 5; // Local variable
              }
        
              hoist();
        
              console.log(a); // 20, global variable
              console.log(b); // ReferenceError: b is not defined

Note: While variables declared with var can be accessed before initialization (due to hoisting), variables declared with let and const cannot be accessed before initialization, making them safer to use to avoid unintended bugs.
