Prototypes in JavaScript

JavaScript is a prototype-based language, which means every object has a built-in property called a prototype. This property allows objects to inherit properties and methods from other objects, facilitating a form of inheritance in JavaScript.

Key Points about Prototypes

  • Prototype is used to inherit properties and methods from one object to another.
  • When accessing a property or method on an object, JavaScript first checks the object itself, and if it’s not found, it looks at the object’s prototype, and then up the prototype chain until it reaches null.
  • The prototype object is essentially a blueprint for other objects, allowing them to share behaviors (methods) without duplicating code.

Ways to Use Prototypes in JavaScript

JavaScript uses prototypes in several different ways:

  • Object Prototype
  • Function Prototype
  • Constructor Prototype
  • Class Prototype

Prototype Chain Example

Here’s an example of how JavaScript's prototype chain works:

            
              function Person(name) {
                  this.name = name;
              }
        
              Person.prototype.greet = function() {
                  console.log("Hello, " + this.name);
              }
        
              let person1 = new Person("Alice");
              person1.greet(); // Output: Hello, Alice
        
              // The 'greet' method is not defined on 'person1' itself, but it's found in Person's prototype.

This demonstrates how JavaScript looks up the prototype chain to find methods or properties that aren’t directly on the object.
