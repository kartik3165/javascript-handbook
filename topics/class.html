Classes in JavaScript

In JavaScript, classes are a blueprint for creating objects. They encapsulate data and methods that define the behavior of the objects. Classes offer a more convenient syntax for creating objects compared to traditional constructor functions, although under the hood, classes are still functions.

Basic Class Example

            
              // Defining a simple Point class
              class Point {
                  constructor(x, y) {
                      this.x = x;
                      this.y = y;
                      console.log("Constructor called:", this); // Point object
                  }
        
                  toString() {
                      return `(${this.x}, ${this.y})`;
                  }
              }
        
              let p = new Point(10, 20);
              console.log(p.toString()); // (10, 20)
              console.log(p instanceof Point); // true

The Point class defines a constructor that initializes an object's x and y properties and includes a method toString that returns a string representation of the point.

Class Inheritance Example

            
              // Extending the Point class to create a ColorPoint class
              class ColorPoint extends Point {
                  constructor(x, y, color) {
                      super(x, y); // Calls the constructor of the parent class (Point)
                      this.color = color;
                      console.log("After super:", this); // ColorPoint object
                  }
        
                  toString() {
                      return `${super.toString()} in ${this.color}`;
                  }
              }
        
              const cp = new ColorPoint(25, 8, 'green');
              console.log(cp.toString()); // (25, 8) in green
              console.log(cp instanceof ColorPoint); // true
              console.log(cp instanceof Point); // true

Here, the ColorPoint class extends the Point class using the extends keyword. The super() call in the constructor allows the child class to access the parent class's constructor and methods. The ColorPoint class overrides the toString method but can still access the parent's toString method using super.toString().

Key Features of Classes

  • Classes provide a cleaner syntax for defining constructor functions.
  • They support inheritance through the extends keyword.
  • Methods defined in classes are not enumerable.
  • Inside a constructor, super() must be called before accessing this in a subclass.
  • Classes can be extended to create more specific types of objects using inheritance.

In this example, cp is both an instance of ColorPoint and Point, showing the power of inheritance in JavaScript classes.
