Difference Between == and === in JavaScript

In JavaScript, == and === are comparison operators, but they behave differently when comparing values. The primary difference is how they handle type coercion.

1. Loose Equality (==)

== allows type conversion when comparing values of different types.

            
              let a = 5;
              let b = '5';
              console.log(a == b);  // true

When using ==, JavaScript converts the string '5' to the number 5 before comparing. This is called type coercion, which results in true.

2. Strict Equality (===)

=== does not allow type conversion and requires both value and type to be identical.

            
              let x = 5;
              let y = '5';
              console.log(x === y);  // false

With ===, JavaScript compares both the value and the type. Since x is a number and y is a string, the result is false because the types don't match.

3. Example Breakdown

Loose Equality Example:

            
              let val1 = 0;
              let val2 = '0';
              console.log(val1 == val2);  // true

In this case, JavaScript converts the string '0' to the number 0 before comparing, resulting in true.

Strict Equality Example:

            
              let val1 = 0;
              let val2 = '0';
              console.log(val1 === val2);  // false

With ===, JavaScript does not perform type conversion. Since val1 is a number and val2 is a string, the comparison returns false.

4. Key Differences

  • Loose Equality: == allows type coercion, making it less strict.
    •               
                let a = null;
                let b = undefined;
                console.log(a == b);  // true
  • Strict Equality: === does not perform type conversion, making it more strict.
    •               
                let a = null;
                let b = undefined;
                console.log(a === b);  // false

5. Best Practice

It is generally recommended to use === over == to avoid unexpected results caused by type coercion. This makes your code more predictable and easier to debug.

            
              let a = '10';
              let b = 10;
        
              if (a === b) {
                  console.log('Strict equality is true');
              } else {
                  console.log('Strict equality is false');  // This will execute
              }
